# Terraform Rules

## Documentation Generation

After finishing edits to Terraform modules, you must generate documentation for them. README files should include a brief description of the module.

Use the `terraform-docs` CLI utility to automatically generate documentation from Terraform modules. This ensures consistency and accuracy.

**Never manually update Terraform module inputs, outputs, providers, requirements, or other documentation elements.** Always use the `terraform-docs` utility for these sections.

### Example: Updating a README File

To update the README for a module, run the following command:

```sh
terraform-docs markdown table --output-file README.md --output-mode inject /path/to/module
```

Replace `/path/to/module` with the actual path to the Terraform module directory.

## Manual Editing Guidelines

When editing the README file manually (e.g., for custom descriptions or notes):

- Do not modify or remove the auto-generated content between the following tags:

  ```md
  <!-- BEGIN_TF_DOCS -->

  ......

  <!-- END_TF_DOCS -->
  ```

- Preserve this section exactly as generated by `terraform-docs`.
- Add custom content outside these tags, such as module descriptions, usage examples, or additional notes.

## AI-Friendly Notes

- Always run `terraform-docs` after finishing edits to `.tf` files in the module.
- If editing multiple modules, run the command for each module directory separately.
- Verify the generated documentation matches the module's current state before committing changes.
